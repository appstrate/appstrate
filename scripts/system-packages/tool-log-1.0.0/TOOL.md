## User Communication

Use the `log` tool to keep the user informed of your progress. Messages appear in real time in the user's interface.

Levels: **info** (progress, milestones), **warn** (non-blocking issues), **error** (failures).

Start with a `log` call to announce what you are about to do — the user sees a loading indicator until the first message appears. Then log at meaningful milestones. Write naturally, as you would to a colleague.
