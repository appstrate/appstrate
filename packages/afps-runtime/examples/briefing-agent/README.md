# briefing-agent — AFPS example bundle

A complete, runnable AFPS bundle that exercises the five CLI commands end-to-end.

## Layout

```
briefing-agent/
├── source/
│   ├── manifest.json     # hand-edited AFPS manifest
│   └── prompt.md         # Mustache template with sections + inverted sections
├── build.ts              # zips source/ and signs → agent.afps
├── agent.afps            # signed bundle (generated)
├── key.json              # Ed25519 keypair — EXAMPLE ONLY, regenerated by build.ts
├── trust-root.json       # trust root pinned to key.json
├── events.json           # 7 scripted RunEvents for deterministic replay
├── context.json          # ExecutionContext (runId, input)
└── snapshot.json         # { memories?, history?, state? } merged onto the context
```

## Regenerate

```sh
bun run examples/briefing-agent/build.ts
```

Rotates the key pair and produces fresh `agent.afps`, `key.json`, `trust-root.json`.

## Try it

From `packages/afps-runtime/`:

```sh
E=examples/briefing-agent

# What's inside?
bun bin/afps.ts inspect $E/agent.afps

# Is it well-formed and signed?
bun bin/afps.ts verify $E/agent.afps --trust-root $E/trust-root.json

# What will the agent actually see?
bun bin/afps.ts render $E/agent.afps \
  --context $E/context.json \
  --snapshot $E/snapshot.json

# Contract-test the full conformance suite (not this bundle specifically)
bun bin/afps.ts conformance
```

### Replay the scripted events deterministically

The old `afps test` wrapper is gone — drive the sink + reducer contract
from a 10-line bun script instead:

```ts
// replay.ts
import { readFile } from "node:fs/promises";
import {
  reduceEvents,
  ConsoleSink,
  FileSink,
  CompositeSink,
  type RunEvent,
} from "@appstrate/afps-runtime";

const events: RunEvent[] = JSON.parse(await readFile("events.json", "utf-8"));
const sink = new CompositeSink([new ConsoleSink(), new FileSink({ path: "/tmp/brief.jsonl" })]);
for (const ev of events) await sink.handle(ev);
await sink.finalize(reduceEvents(events));
```

Run with `bun replay.ts`.

## Execute against a real LLM

Live execution is handled by the `appstrate run` command (in
`apps/cli`), which shares this runtime as a workspace dependency. Same
PiRunner, same sinks, plus profile / credential / telemetry support
when an Appstrate instance is pinned. `--providers=none` and
`--report=false` keep it fully offline — the briefing-agent bundle
uses no authenticated providers, so this example works without any
platform in the loop.

```sh
export ANTHROPIC_API_KEY=sk-...

bun apps/cli/src/cli.ts run $E/agent.afps \
  --providers=none --report=false \
  --model-source=env --model-api=anthropic-messages \
  --model=claude-haiku-4-5-20251001 \
  --snapshot $E/snapshot.json \
  --input "$(jq -c .input $E/context.json)" \
  --run-id "$(jq -r .runId $E/context.json)"
```

## What this demonstrates

- **Mustache sections** — `{{#memories}}…{{/memories}}` expands the two
  memories from `snapshot.json`; `{{^state}}…{{/state}}` would trigger
  if `state` were absent (it isn't, so the `{{#state}}…{{/state}}`
  branch fires instead).
- **Event reducer** — run produces `memories=1` (one `add_memory`
  event), `state=set` (last-write-wins), `output=set` (merge-patch of
  two partial outputs), `report=set` (final summary), `logs=2`.
- **Signature round-trip** — `verify` checks the `signature.sig`
  against the `trust-root.json` pinned to the Ed25519 public key in
  `key.json`.
- **Deterministic replay** — every run is byte-identical given the
  same `events.json` / `context.json` / `snapshot.json`.

The bundle is realistic enough to show the shape of a production agent,
but shipped without any LLM dependency — a ten-line `replay.ts` (see
above) feeds `events.json` straight through an `EventSink` +
`reduceEvents`, exactly the way a real `Runner` implementation would
stream its events at execution time.
